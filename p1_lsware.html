프로젝트 요약

중소기업 인턴 근무 중에 서버 취약점 분석 SW제품의 QA자동화 프로그램을 개발했습니다. Linux, Solaris, Window 등 서버OS 별 기본 보안 설정(유저권한, 원격접속 등)을 점검하고 SW제품의 분석결과와 비교하여 보다 신속하게 QA결과를 도출했고 제품 정확성을 높였습니다.

  • 개발내용 : QA 자동화 프로그램 개발
  • 기간 : 2018.04 - 2018.06
  • 서비스 대상 : 사내 QA부서 담당자


    • 인력구성 및 역할

  • 프로젝트 인원 : 1人 프로젝트
  • SW제품 주요 기능 및 OS별 보안 환경 분석
  • 요구사항 분석서, WBS, 화면설계서 등 프로젝트 문서 작성
  • 서버 보안설정 값을 분석하여 SW제품 정확성을 도출하는 로직 개발


    • 상세 수행 내용

  • 서버OS별 예외 상황(응답값 포멧 등)에 따른 처리 로직 개발
  • 대상 서버로 Telnet 접속하여 테스트 케이스 별 보안설정 점검 로직 개발
    ➡️   root권한으로 접근하여 서버 환경설정 파일 확인 및 명령어로 확인
    ➡️   서버 응답값(파일내용/명령어 결과값)의 문자열 비교를 통해 설정여부 확인
  • SW제품의 점검결과와 QA프로그램 결과를 비교한 Report생성(Export)

    • Tools Used

    Python
    LSWare SW(QA 대상 제품)
